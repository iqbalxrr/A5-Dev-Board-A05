Question-2: What is the difference between innerHTML, innerText, and textContent ?

innerHTML:
  • If we want to set innerHTML, it replaces the entire HTML content inside the element with the new markup (including tags).
  • It does not consider visibility or styling of elements.
  • Purpose: if we want to gets or sets HTML tags inside an element .

innerText:
  • it's not showing include HTML tags
  • it's show the visible content(hidden text not showing ).
  • whitespace adjusts( ignores excess spaces and line breaks ).
  • Purpose: gets or sets the visible text content inside an element

textContent:
  • it's not showing include HTML tags
  • It does not consider visibility (it returns all text, including hidden elements).
  • It preserves whitespace formatting but ignores extra spaces or line breaks.
  • Purpose: get or sets the text content of an element, excluding any HTML markup

Question-3: What is event delegation in the DOM?

Event delegation is a technique in JavaScript that allows you to handle events for multiple elements in a more efficient way. Instead of attaching event listeners to each individual element, you can attach a single event listener to a parent element and use event delegation to handle events on its child elements.

Question-4: What is event bubbling in the DOM?

Event bubbling is a concept in the Document Object Model (DOM) where events propagate from the most specific element to the least specific element. When an event occurs on a specific element, it first triggers the event on that element, and then it bubbles up to its parent elements, and so on, until it reaches the root element (usually the document object).
